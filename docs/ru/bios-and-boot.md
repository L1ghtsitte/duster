# BIOS, сеть и автозагрузка станций

Чтобы Duster мог **включать ПК удалённо** и стабильно подключать агенты после перезагрузки, настройте железо и Windows один раз на каждой станции.

## Wake-on-LAN (включение по сети)

### BIOS / UEFI

Включите (названия могут отличаться):

- **Wake on LAN** / **Power On By PCI-E** / **PME Event Wake Up** → Enabled
- **ErP** / **Deep Sleep** → Disabled (мешает WoL на некоторых платах)
- После настройки: **сохранить и выключить** ПК (не перезагрузка - именно shutdown, чтобы NIC оставался под питанием)

### Сетевой адаптер (Windows)

Диспетчер устройств → Сетевые адаптеры → свойства NIC:

| Вкладка | Параметр | Значение |
|---------|----------|----------|
| Дополнительно | Wake on Magic Packet | Enabled |
| Дополнительно | Wake on Pattern Match | по желанию |
| Управление питанием | Разрешить устройству выводить компьютер из ждущего режима | ✓ |
| Управление питанием | Разрешить только magic packet… | ✓ (рекомендуется) |

Запишите **MAC-адрес** этой NIC в админке (формат `AA:BB:CC:DD:EE:FF`).

### Сеть клуба

- WoL работает в **той же подсети** (L2 broadcast). Между VLAN нужен relay или directed broadcast.
- UDP порт **9** (можно 7) не блокировать на коммутаторе.
- Сервер Duster шлёт пакет на `255.255.255.255`; при необходимости укажите broadcast подсети в коде (`sendWakeOnLan`).

Проверка: админка → ПК → **WoL** → ПК должен начать загрузку за 10–60 сек.

## Автозагрузка в оболочку Duster

### Вариант A: Автовход + служба агента

1. `Win+R` → `netplwiz` → отключить «требовать ввод пароля» для учётки `club`.
2. Служба **DusterAgent** → Тип запуска: **Автоматически**.
3. При старте Windows агент подключается к серверу; статус ПК → `online` или `locked`.

### Вариант B: Shell вместо explorer (жёсткий киоск)

> Осторожно: неправильная настройка лишит доступа к рабочему столу. Делайте образ диска / точку восстановления.

```reg
[HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows NT\CurrentVersion\Winlogon]
Shell = "C:\Duster\shell.exe"
```

`shell.exe` - ваша оболочка (или запуск `msedge --kiosk http://server:5174?pc=1`).

Для Gizmo-подобного режима «липкая оболочка»:

- Отключить Task Manager / Alt+Tab политиками GPO.
- Запретить смену пользователя.

### Вариант C: Assigned Access (Windows 10/11 Pro+)

Параметры → Учётные записи → Назначенный доступ → выбрать UWP/браузер в киоске.

## Загрузка с BIOS / PXE (diskless)

Duster **не заменяет** PXE/iSCSI-стек. Типичная схема diskless-клуба:

1. BIOS: **Network Boot** первым (только для бездисковых).
2. Сервер образов (ggRock, CCBoot и т.д.) отдаёт Windows.
3. На образе предустановлены: агент Duster, `appsettings` с уникальным токеном **на образе** или выдача токена по hostname при первом старте (доработка).

Для **дисковых** ПК: загрузка с SSD, в BIOS отключить boot с USB, пароль на BIOS - защита от обхода.

## Выключение и блокировка

| Команда Duster | Действие Windows |
|----------------|------------------|
| lock | `LockWorkStation` |
| shutdown | `shutdown /s` |
| restart | `shutdown /r` |
| unlock | сигнал агенту; сессия активна, кассир/игрок разблокирует вход |

После `shutdown` для повторного включения снова нужен **WoL** или физическое включение.
