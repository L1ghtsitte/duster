# Duster - установка по компонентам

Пошаговые инструкции для каждой части системы. Выполняйте в порядке: **сервер → админ → агенты → оболочка**.

---

## 0. Общие требования

| Компонент | Требование |
|-----------|------------|
| Сервер / админ | Node.js **20+**, npm |
| Игровые ПК | Windows 10/11, .NET **8 Runtime** |
| Сеть | Статический IP сервера в LAN, порт **3847** открыт |

Первичная настройка из корня проекта:

```powershell
cd c:\Users\kek\Documents\duster
.\scripts\setup.ps1
```

---

## 1. Сервер (центральный ПК или VM)

**Где:** отдельный ПК или тот же, что админ, но служба всегда включена.

1. Скопируйте проект на машину, например `C:\Duster`.
2. В `packages\server\.env` задайте:
   - `DUSTER_JWT_SECRET` - длинная случайная строка
   - `DUSTER_PORT=3847`
3. Инициализация БД:
   ```powershell
   cd packages\server
   npx prisma db push
   npm run db:seed
   ```
4. Запуск для проверки: `npm run dev` из `packages\server`.
5. Проверка: браузер → `http://IP:3847/api/health` → `{"ok":true}`.
6. **Продакшен - служба Windows (NSSM):**
   ```powershell
   nssm install DusterServer "C:\Program Files\nodejs\node.exe"
   nssm set DusterServer AppParameters "C:\Duster\packages\server\dist\index.js"
   nssm set DusterServer AppDirectory C:\Duster\packages\server
   nssm start DusterServer
   ```
7. Брандмауэр: входящий TCP **3847** только для подсети клуба.

**Данные:** PDF и файлы смен - `packages\server\data\reports` и `data\shifts`.

---

## 2. Админ-панель (касса / главный ПК)

**Где:** ПК администратора с браузером.

1. Убедитесь, что сервер запущен.
2. Разработка:
   ```powershell
   npm run dev:admin
   ```
   Откройте http://localhost:5173 - логин `admin` / `admin123` (смените после входа).
3. **Продакшен:** соберите и раздавайте статику:
   ```powershell
   npm run build --workspace=@duster/admin-web
   ```
   Настройте nginx/IIS: `/` → `admin-web\dist`, `/api` и `/ws` → прокси на `http://SERVER:3847`.

**Смена кассира:** меню **Смены** → открыть смену (касса на начало) → в конце **Закрыть** → скачать PDF и `.dshift.json`.

---

## 3. Бонусы к пополнению

**Где:** админка → **Бонусы** (или API `/api/admin/settings/topup-tiers`).

1. Добавьте уровни: «от 500₽ → 5%», «от 1000₽ → 10%» и т.д.
2. При пополнении в **Профиле игрока** виден превью бонуса.
3. **Персональный %:** профиль игрока → поле «Персональный бонус %».
4. **Безлимит:** профиль → «Безлимитное время».
5. **Минуты вручную:** профиль → «Начислить минуты» (без оплаты).

---

## 4. Профили игроков

**Где:** админка → **Игроки** → **Профиль**.

- Контакты, PIN, группа (standard/vip/staff/guest).
- История пополнений и сессий.
- Пополнение с автоматическим %.

**Станция:** игрок входит PIN на веб или WPF - см. п. 6–7.

---

## 5. Windows-агент (каждый игровой ПК)

**Зачем:** lock/unlock, выключение, скриншот для удалённого просмотра.

1. Админка → **Компьютеры** → добавьте ПК → скопируйте **токен агента**.
2. `apps\client-agent\Duster.Agent\appsettings.json`:
   ```json
   "ServerUrl": "http://192.168.10.10:3847",
   "AgentToken": "ВАШ_ТОКЕН"
   ```
3. Сборка:
   ```powershell
   cd apps\client-agent\Duster.Agent
   dotnet publish -c Release -r win-x64 --self-contained false
   ```
4. Установка службы (cmd от администратора):
   ```powershell
   sc create DusterAgent binPath= "C:\Duster\agent\Duster.Agent.exe" start= auto
   sc start DusterAgent
   ```
5. В админке статус агента: **агент ✓**, статус `online` / `locked`.

Подробно BIOS/WoL: [bios-and-boot.md](./bios-and-boot.md).

## 6. Веб-оболочка станции

**Когда:** `shellMode = web` в карточке ПК.

1. На сервере/прокси: `npm run dev:client-web` или build `client-web`.
2. На станции ярлык:
   ```
   msedge.exe --kiosk http://192.168.10.10:5174?pc=3
   ```
   `pc=3` - номер ПК из админки.
3. Игрок вводит PIN → API `POST /api/station/login` → сессия, разблокировка через агент.

---

## 7. WPF-оболочка (native UI)

**Когда:** `shellMode = native`, нужен вид «как приложение», не браузер.

1. Создайте `C:\Duster\server.url` с URL сервера.
2. Переменная среды `DUSTER_PC=3` (номер станции).
3. Сборка:
   ```powershell
   cd apps\client-shell\Duster.Shell
   dotnet publish -c Release
   ```
4. **Замена оболочки Windows** (осторожно, сделайте образ диска):
   - `Win+R` → `regedit`
   - `HKLM\SOFTWARE\Microsoft\Windows NT\CurrentVersion\Winlogon` → `Shell` = `C:\Duster\shell\Duster.Shell.exe`
5. Агент (п. 5) обязателен для lock и скриншота.

---

## 8. Удалённый просмотр экрана

**Где:** админка → **Компьютеры** → **Экран**.

1. Агент на ПК должен быть онлайн.
2. Нажмите **Экран** - сервер запросит `screenshot` у агента.
3. Для обновления используется WebSocket (`watch_screen`).

Ограничение: снимок экрана, не полноценный RDP; для плавного видео - будущая версия WebRTC.

---

## 9. Смены, PDF и файлы `.dshift`

1. Кассир: **Смены** → **Открыть смену** (сумма в кассе).
2. Все продажи с этой кассы привязываются к смене.
3. **Закрыть смену** → ввод кассы на конец → автоматически:
   - PDF в `packages\server\data\reports\`
   - JSON `.dshift` в `data\shifts\`
4. Скачать: кнопки **PDF** / **.dshift** в истории.
5. **Просмотр старой смены:** вставить JSON или загрузить `.dshift` в блок импорта.

---

## 10. Wake-on-LAN и питание

1. Внесите **MAC** в карточку ПК.
2. Настройте BIOS/NIC - [bios-and-boot.md](./bios-and-boot.md).
3. Админка → **WoL** - включение выключенного ПК.
4. **Выкл / Блок** - команды агенту.

## 12. Устранение неполадок

| Проблема | Решение |
|----------|---------|
| Агент ✗ | Проверьте token, URL, брандмауэр, службу |
| PIN не входит | PIN в профиле игрока, ПК `number` совпадает с `?pc=` |
| Нет PDF | Папки `data/reports`, права записи, закрывайте смену через UI |
| Нет бонуса | Уровни в **Бонусы** или персональный % в профиле |
| Чёрный экран | Агент под учёткой с доступом к рабочему столу |

Архитектура: [architecture.md](./architecture.md).
